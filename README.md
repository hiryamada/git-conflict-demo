# Git コンフリクト・マージ演習

mainブランチの最初のコード

```python
def greet(s):
    print(f"こんにちは, {s}!")
    
if __name__ == "__main__":
    greet("Taro")
```

## やりたいこと

- メッセージの英語化 (english ブランチで実施)
- パラメータ名 s を name に改善 (param ブランチで実施)

## 準備

- このリポジトリをローカルにクローン
- クローンしたコードを Visual Studio Code で開く
- app.py を開く
- 「ソース管理」を開く

## 操作例（コンフリクトが起きるまで）

- english ブランチを作成し、切り替え
- greet 関数の日本語のメッセージを英語化
- ファイルを保存、コミット
- main ブランチに切り替え
- param ブランチを作成し、切り替え
- パラメータ名 s を name に変更
- ファイルを保存、コミット
- main ブランチに切り替え
- main ブランチに english ブランチをマージ
- main ブランチに param ブランチをマージ（ここでコンフリクトが発生）

## コンフリクトが起こった理由

printf の行に対し、2つのブランチで同時に変更が行われているため。

## 操作例（コンフリクトの解決・マージ）

- 「マージエディターで解決」をクリック
- 画面右上「現在のマシンを適用する」をクリック（英語化される）
- 画面左上「組み合わせを承諾する」をクリック（パラメータ名が改善される）
- 画面下「マージの完了」をクリック
- 「ソース管理」で「続行」をクリック
